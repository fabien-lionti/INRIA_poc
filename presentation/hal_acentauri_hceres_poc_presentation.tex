\begin{frame}[t]
  \SlideTitle{1. HAL ACENTAURI HCERES POC}

  \large
  A reproducible bibliometric pipeline to support HCERES-style evaluation preparation.

  \vspace{0.8em}

  \begin{block}{Main message}
    The goal is not to automate the evaluation itself, but to generate a verifiable working basis for expert validation.
  \end{block}

  \begin{itemize}
    \item Context: ACENTAURI scientific production indexed in HAL.
    \item Goal: produce deterministic tables, figures, and report artifacts.
    \item Constraint: keep every result auditable and rerunnable.
  \end{itemize}

  \vfill
  \scriptsize Inria / ACENTAURI -- Summary presentation -- \today
\end{frame}

\begin{frame}[t]
  \SlideTitle{2. Problem Addressed}

  \begin{block}{Evaluation preparation is data-heavy}
    HCERES-style preparation requires a reliable synthesis of publications, collaborations, venues, open-science indicators, and validation gaps.
  \end{block}

  \vspace{0.3cm}

  \begin{columns}[T,onlytextwidth]
    \begin{column}{0.50\textwidth}
      \textbf{Current bottlenecks}
      \begin{itemize}
        \item HAL metadata is available but heterogeneous.
        \item Manual extraction is repetitive and slow.
        \item Inconsistencies are hard to detect afterwards.
        \item Missing information is often mixed with verified information.
      \end{itemize}
    \end{column}

    \begin{column}{0.46\textwidth}
      \textbf{Expected support}
      \begin{itemize}
        \item Produce a first clean working basis.
        \item Separate computed, inferred, manual, and missing data.
        \item Make the validation workload explicit.
        \item Keep outputs reproducible.
      \end{itemize}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}[t]
  \SlideTitle{5. Initial Prompt to the AI}

  \small

  \begin{block}{Reconstructed specification}
    \emph{Create a simple, monolithic Python proof of concept that automates a HAL-based bibliometric analysis of the ACENTAURI team for HCERES-style evaluation preparation.}
  \end{block}

  \begin{itemize}
    \item Query HAL automatically, with structure discovery or explicit override.
    \item Clean, filter, and deduplicate publication records.
    \item Compute indicators on output, authors, document types, collaborations, venues, and open science.
    \item Accept optional CSV files as enrichments without making them mandatory.
    \item Generate deterministic reports with explicit provenance and no invented content.
  \end{itemize}

  \vspace{0.4em}

  \scriptsize
  The exact prompt was not versioned; this slide summarizes the specification reflected by the generated code.

\begin{frame}[t]
  \SlideTitle{12. Narrated Video Synchronization}

  \begin{block}{Goal}
    Make the narrated video easier to follow by aligning each slide duration with its spoken explanation.
  \end{block}

  \begin{enumerate}
    \item Each slide has a dedicated narration paragraph.
    \item The TTS voice is slowed down to improve comprehension.
    \item Each slide clip duration is computed from the generated audio.
    \item Long TTS silences are compressed to avoid empty waiting time.
    \item A title card is inserted before the slide sequence.
  \end{enumerate}

  \begin{block}{AI support}
    Codex created the video-generation script, adjusted the voice speed, compressed silences, and regenerated the final MP4.
  \end{block}
\end{frame}

\begin{frame}[t]
  \SlideTitle{13. How This Video Was Generated}

  \begin{block}{Generation workflow}
    The final MP4 was generated automatically from the Beamer PDF and the narration script.
  \end{block}

  \begin{enumerate}
    \item Compile the Beamer source to PDF with \texttt{pdflatex}.
    \item Render each PDF page as a PNG image with \texttt{pdftoppm}.
    \item Generate one audio narration per slide with \texttt{edge-tts}.
    \item Slow down the TTS voice to improve clarity.
    \item Compress long TTS silences with \texttt{ffmpeg}.
    \item Create one still-image video clip per slide.
    \item Match each clip duration to its narration audio.
    \item Concatenate all clips into the final MP4.
  \end{enumerate}
\end{frame}
